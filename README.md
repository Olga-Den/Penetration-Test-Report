## windows-linux-epp-antivirus-detection-testing
Testowanie skuteczności systemów EPP (Endpoint Protection) na Windows 10/11 (Microsoft Defender) oraz Linux Server (ClamAV) z wykorzystaniem pliku testowego EICAR oraz próbek malware dostarczonych w archiwach chronionych hasłem. Porównano detekcję lokalną (sygnaturową) z wynikami analizy w VirusTotal (70+ silników AV + sandbox behawioralny), wykazując różnicę między ochroną on-access (Windows Defender) a on-demand (ClamAV). Zweryfikowano również zdolność ClamAV do wykrywania malware pisanego pod Windows na serwerach Linux pełniących rolę bramy plikowej/pocztowej. <br/>
Narzędzia: Microsoft Defender, ClamAV (clamscan), VirusTotal, 7-Zip, EICAR test file.
## iptables-windows-firewall-hardening-nmap
Konfiguracja i testowanie reguł firewalla systemowego na iptables (Ubuntu Server) oraz Windows Defender Firewall (PowerShell) w środowisku z maszyną atakującą Kali Linux. Wdrożono politykę Default Drop z logowaniem zdarzeń, a następnie reguły blokujące m.in. nieautoryzowany dostęp do MariaDB (port 3306), pakiety TCP o nieprawidłowym stanie, ruch UDP 1024-20000, ICMP poza ping, skanowanie typu Christmas Tree, ataki brute-force SSH oraz rate-limiting ICMP Echo (moduł recent). Skuteczność zweryfikowano za pomocą Nmap (SYN/UDP/XMAS scan) i analizy logów (syslog, Windows Firewall log). <br/>
Narzędzia: iptables, Windows Defender Firewall (New-NetFirewallRule), Nmap, Kali Linux.
## suricata-nids-nmap-misp-detection-lab
Wdrożenie i konfiguracja Suricata na Ubuntu Server w trybach Sniffer, Packet Logger oraz NIDS. Utworzono własne reguły detekcji (local.rules) wykrywające skanowanie portów, SYN Flood, próby logowania uprzywilejowanych kont SSH, nietypowe kombinacje flag TCP oraz wstrzykiwanie kodu JavaScript. Skuteczność zweryfikowano za pomocą Nmap (SYN/XMAS/FIN/NULL scan) i skryptów NSE (http-vuln-cve2010-2861, smb-vuln-regsvc-dos, ssh-brute) z maszyny Kali Linux, analizując alerty w fast.log i eve.json. Rozszerzono detekcję o zewnętrzne źródła reguł (Emerging Threats Open, Abuse.ch URLHaus) przez suricata-update oraz omówiono integrację z MISP.<br/> 
Narzędzia: Suricata, Nmap + NSE, tcpdump, suricata-update, MISP, Kali Linux.
## pfsense-idps-vlan-segmentation-suricata-report
Zbudowanie środowiska IDPS na bazie pfSense z segmentacją sieci na 8 VLAN (ADMIN, USERS, SERVERS, DMZ, PRINTERS, BACKUP, CCTV, QUARANTINE), konfiguracja IDS/IPS Suricata (reguły Emerging Threats, Snort GPLv2, Feodo Tracker) z automatycznym blokowaniem ataków (Block Offenders) oraz filtrowaniem złośliwych domen i IP przez pfBlockerNG (DNSBL). Środowisko testowe (hosty, serwery, DMZ) odwzorowano w kontenerach Docker, a ataki - skanowanie portów, brute-force SSH, skanowanie aplikacji webowej - przeprowadzono z Kali Linux i wykryto na podstawie alertów Suricata oraz logów EVE JSON. Dodatkowo skonfigurowano bezpieczny zdalny dostęp przez OpenVPN (certyfikaty, TLS Auth, AES-GCM/ChaCha20). <br/>
Narzędzia: pfSense, Suricata, pfBlockerNG, OpenVPN, Docker, Nmap, Hydra, Nikto, Kali Linux.
## metasploitable2-manual-vulnerability-detection-report
Ręczne wykrywanie i weryfikacja 9 podatności w systemie Metasploitable2 bez użycia automatycznych skanerów, z eliminacją False Positives poprzez bezpośrednią interakcję z każdą usługą. Zidentyfikowano i potwierdzono m.in. backdoor vsftpd 2.3.4 (CVE-2011-2523), przechwycenie w plaintext danych logowania Telnet, zdalne wykonanie kodu przez distccd (CVE-2004-2687), dostęp roota bez hasła do MySQL i PostgreSQL oraz bindshell root na porcie 1524. Każdą podatność opisano w tabeli z oceną CVSS, metodą wykrycia i sugerowaną mitygacją.<br/>
Narzędzia: Nmap, Netcat, tcpdump, Hydra, dig, MySQL client, psql, rlogin, VNC viewer.
## openvas-nessus-vulnerability-scanning-comparison-report
Automatyczne skanowanie podatności systemu Metasploitable2 dwoma skanerami - OpenVAS/Greenbone oraz Nessus Essentials - z pełną analizą wszystkich portów TCP. Wykryto krytyczne podatności (CVSS 10.0), w tym backdoory UnrealIRCd i vsFTPd, podatność Ghostcat w Apache Tomcat AJP oraz liczne problemy konfiguracji SSL/TLS (SSLv2/v3, słabe szyfry). Przeprowadzono analizę ruchu sieciowego generowanego przez oba skanery w Wireshark (banner grabbing) oraz porównano skuteczność wykrywania, klasyfikację CVSS i jakość raportowania obu narzędzi. <br/>
Narzędzia: OpenVAS (Greenbone), Nessus Essentials, Wireshark, Kali Linux, Metasploitable2.
## metasploit-metasploitable2-pentest-report
Przeprowadzenie kontrolowanego testu penetracyjnego na maszynie Metasploitable2 przy użyciu Metasploit Framework. Zidentyfikowano i wykorzystano metodą PoC 8 podatności (m.in. vsFTPd 2.3.4 Backdoor, Samba Usermap Script, UnrealIRCd Backdoor, DistCC, Apache Tomcat Manager, PostgreSQL, Java RMI, NFS), uzyskując zdalne powłoki/sesje Meterpreter z uprawnieniami root. Następnie wdrożono i zweryfikowano działania naprawcze (wyłączenie podatnych usług), potwierdzając ich skuteczność ponowną próbą ataku.<br/>
Narzędzia: Kali Linux, Metasploit Framework, Metasploitable2, OpenVAS/Nessus.
## zabbix-monitoring-incident-simulation-report
Wdrożenie systemu monitoringu Zabbix (serwer + agent) w środowisku dwóch maszyn wirtualnych oraz przeprowadzenie testów stresowych: obciążenie CPU/RAM, krytyczne zapełnienie dysku, twardy restart systemu i awaria agenta. Skonfigurowano hosty, szablony i triggery, a następnie zweryfikowano poprawność detekcji anomalii w czasie rzeczywistym.<br/>
Narzędzia: Zabbix Server/Agent 7.0, VirtualBox/VMware, Linux, SQL, SSH.
## siem-elastic-atomic-red-team-detection-report
Zbudowanie środowiska SIEM na bazie Elasticsearch i Kibana z telemetrią z hosta Windows (Sysmon + Winlogbeat), a następnie symulacja 4 technik ataku wg MITRE ATT&CK (T1047 WMI Recon, T1003.001 LSASS Dump, T1059.001 PowerShell Encoded Command, T1136.001 Create Account) przy użyciu Atomic Red Team. Wszystkie ataki wykryto i przeanalizowano za pomocą zapytań KQL w Kibana.<br/>
Narzędzia: Elasticsearch, Kibana, Sysmon, Winlogbeat, Atomic Red Team, PowerShell.
